# 项目总览

## 项目名称
Amazon运营推进器（原项目名称：电商运营管理系统）

## 当前阶段
当前阶段已经不再是早期“待办事项模块工程化启动”阶段，而是进入了：

**首页工作台重构 + 模块3高保真前端重构 + 新对话可持续接手阶段**

当前开发重点明确是：
- 优先继续前端页面
- 优先保证老板确认过的页面结构与展示逻辑
- 数据库与后端真实联调暂不作为当前第一优先级

## 当前聚焦范围
当前系统主页已重构为 3 个入口：
1. 俄罗斯WB运营（占位页）
2. 项目管理进度（占位页）
3. Amazon运营推进器（当前主开发页面）

Amazon运营推进器首页当前实际聚焦 3 个模块：
- 模块1：竞品上新信息
- 模块2：竞品监控
- 模块3：运营数据主卡区

其中真正的主开发重点仍然是：

**模块3运营数据展示 + 首页工作台整体统一视觉**

## 当前目标
1. 继续精修 Amazon运营推进器首页与模块3视觉
2. 保住模块1 / 模块2 / 模块3已有功能，不在改版中丢失
3. 让新对话只通过文档即可快速接手当前项目状态
4. 后续视页面稳定情况，再决定恢复后端真实权限和数据库联调

## 当前已确认的重要结论
- 系统主名称已统一为 **Amazon运营推进器**
- 左侧入口固定保留 3 个系统：俄罗斯WB运营、项目管理进度、Amazon运营推进器
- 首页顶部采用“姓名，今日工作”形式
- 首页顶部保留模板化摘要卡
- 模块3保留以下业务语义与功能：
  - 评价信息
  - 销售数据
  - 流量数据
  - SP广告
  - SBV广告
  - 折叠展开
  - 已读 / 未读状态
  - 广告活动明细表
- 当前页面改版原则是：
  - **先保功能，再精修样式**
  - **不主动扩数据库**
  - **不主动改后端接口契约**
- 当前前端已恢复“按运营查看本人负责产品”的前端模拟逻辑：
  - 已支持运营切换
  - 已支持按负责人过滤模块1、模块2、模块3数据
  - 当前仍属于前端模拟，不是后端真实权限

## 当前建议优先级
1. 继续精修顶部摘要卡与模块3视觉细节
2. 继续让模块1、模块2卡片骨架更贴近模块3模板风格
3. 再评估是否接入后端真实负责人字段 / 登录态
4. 页面稳定后，再恢复数据库与真实联调工作
