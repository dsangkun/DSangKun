# 12-2026-04-28-数据页日报 Sheet 对接说明

## 一、当前口径

当前数据页仍保持第一阶段策略：

- 一个父 ASIN 数据页对应一个日报 Sheet；
- 前端先直接展示原始日报 Sheet；
- 不把任务页和数据页重新混在一起；
- 首页模块3继续保持“父ASIN卡片 = 任务页入口，数据页独立展示”。

## 二、本次补强内容

### 1）Sheet 匹配从“完全等值”升级为“可诊断匹配”

原先数据页仅按：

- `父ASIN产品名 === Sheet 名`

做直接匹配。这个方式在第一阶段足够快，但脆弱点明显：

- 映射表父产品名与 Sheet 名只要有一点差异，就会打不开；
- 英文长标题 Sheet 常有截断；
- 有些 Sheet 名不等于父产品名，但首列中存在父/子产品名称；
- 排查失败时没有候选信息，定位成本高。

因此本次已改为多级匹配：

1. 父产品名与 Sheet 名完全匹配；
2. 父产品名与 Sheet 名归一化匹配（忽略空格/符号/大小写）；
3. 父产品名命中 Sheet 首列主题；
4. 子产品名命中 Sheet 首列主题；
5. 名称包含关系兜底；
6. 展示匹配置信度、匹配方式、候选 Sheet，便于排查。

### 2）日报 Sheet 常量改为脚本可再生

新增脚本：

- `ops/extract_daily_report_sheets.py`

用于从多个 Excel 文件中批量抽取全部 Sheet，并生成：

- `frontend/src/constants/dailyReportSheets.ts`

这样后续日报日期变化时，不需要再手工维护常量文件，只需要重新执行脚本。

## 三、执行方式

在前端目录执行：

```bash
npm run extract:daily-report-sheets -- "D:\Programs\testdata\EXPERLAM-亚马逊\20260421\EXPERLAM-美国-流量数据统计报表-20260421.xlsx" "D:\Programs\testdata\KHINYA-亚马逊\20260421\KHINYA-亚马逊-流量数据统计报表-20260421.xlsx"
```

说明：

- 可一次传入多个日报 Excel；
- 后传入的同名 Sheet 会覆盖前面的同名 Sheet；
- 输出文件默认覆盖 `frontend/src/constants/dailyReportSheets.ts`。

## 四、现阶段建议

当前阶段不建议立刻切到“运行时后端接口读 Sheet”，原因如下：

1. 当前目标是先把数据页展示稳定下来；
2. 日报来源还是本地 Excel，后端接口化本质上只是把同一份临时原始数据换个出口；
3. 如果过早接口化，会把问题从“展示稳定性”扩散到“文件读取、接口设计、权限、缓存、部署路径”；
4. 当前前端直展示 + 可再生脚本，已经足够支撑下一阶段样式精修和运营对齐。

## 五、下一步优先级建议

1. 继续精修 `ProductTaskDetailView.vue` 与样式，让表格更接近 Excel；
2. 统计当前映射表父产品名与 Sheet 匹配失败的清单，补齐别名策略；
3. 等日报来源和口径稳定后，再决定是否后端接口化；
4. 如果未来需要跨日期切换、历史追溯、权限审计，再引入后端数据接口更合适。
