# 运营数据原始五表口径说明

更新时间：2026-04-02

## 一、每日原始输入

对于单店铺单日，原始输入由以下 5 份数据组成：

1. **业务报告-父ASIN**
   - 粒度：父 ASIN / 日
   - 主要提供：会话、页面浏览、订单、销量、销售额、转化率等

2. **业务报告-子ASIN**
   - 粒度：子 ASIN / 日
   - 主要提供：子体销量、流量、销售额、转化率等

3. **SP广告报告**
   - 粒度：广告明细 / 子 ASIN / SKU / 日
   - 主要提供：曝光、点击、CTR、CPC、花费、7日销售额、ACOS、ROAS 等

4. **SBV汇总**
   - 粒度：子 ASIN / 日
   - 主要提供：SBV 曝光、点击、CTR、CPC、花费、销售额、ACOS、广告订单、CVR

5. **ASIN映射表**
   - 粒度：父 ASIN -> 子 ASIN
   - 主要提供：
     - 父 / 子 ASIN 关系
     - 父 / 子产品命名
     - 是否有 SBV

## 二、最终流量统计报告的本质

`流量数据统计报告.xlsx` 不是原始事实表，而是对上述五张表按产品维度重组后的结果表。

因此它更适合：

- 用作人工核对
- 用作汇总口径校验
- 用作导入结果验算

不适合直接作为数据库主事实来源。

## 三、推荐数据库分层

### 1. 原始层（raw）
一张原始表对应一类来源文件。

### 2. 汇总层（dws / agg）
按产品、父 ASIN、子 ASIN、日期重建业务口径。

### 3. 接口层（api）
为模块3返回：
- 当日值
- 7日均值
- 上周同日值
- 环比变化

## 四、当前最优开发顺序

1. 建原始层表
2. 导入五张原始表
3. 校验与最终流量统计报告是否一致
4. 再写后端聚合查询
5. 最后切前端模块3
