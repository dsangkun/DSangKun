# 后端接口草案（工作台 / 待办事项模块）

## 1. 接口目标

为前端工作台页面提供最小可用接口，覆盖三大板块：
- 竞品上新信息
- 竞品监控变化
- 运营数据对比

当前阶段先返回 Mock 数据结构，保证前后端接口契约先固定下来。

## 2. 接口列表

### 2.1 健康检查
- `GET /api/health`

响应示例：
```json
{
  "success": true,
  "message": "OK",
  "data": {
    "service": "ecommerce-ops-backend",
    "status": "UP"
  }
}
```

### 2.2 工作台概览
- `GET /api/workbench/overview`

### 2.3 竞品上新列表
- `GET /api/workbench/new-arrivals`

### 2.4 竞品上新处理动作
- `POST /api/workbench/new-arrivals/{id}/action`

请求体：
```json
{
  "action": "PUSH"
}
```

支持动作值：
- `PUSH`
- `TRACK`
- `IGNORE`

### 2.5 竞品监控变化列表
- `GET /api/workbench/competitor-changes`

### 2.6 运营数据列表
- `GET /api/workbench/operation-data`

## 3. 返回结构约定

统一返回结构：

```json
{
  "success": true,
  "message": "OK",
  "data": {}
}
```
