# 开发路线与技术路线（电商运营管理系统）

> 目的：把“电商运营管理系统”的开发节奏、技术选型、环境现状与待办事项固化下来，便于后续按阶段推进与验收。

## 1. 项目定位与当前阶段

- 项目名称：电商运营管理系统
- 当前阶段：**需求落地 + M1 前端工程化阶段**
- 当前聚焦模块：**工作台 → 待办事项模块**
- 当前阶段目标：
  1) 固化待办事项模块需求（见 `docs/01-需求清单.md`）
  2) 形成可直接演示的静态页面（见 `frontend/todo-demo.html`）
  3) 将静态 demo 迁移为 Vue 工程结构，便于后续扩展与联调
  4) 暂不接入真实数据、不做接口联调、不做权限体系

## 2. 技术路线（推荐技术栈）

> 选型原则：先可演示、后可扩展；以“稳定、可维护、易部署”为优先。

### 2.1 前端（管理端）

- 推荐：**Vue 3 + TypeScript + Vite**
- UI：Element Plus（或 Ant Design Vue，二选一）
- 状态管理：Pinia
- 路由：Vue Router
- 请求：Axios
- 图表：ECharts
- 代码规范：ESLint + Prettier

> 说明：当前仓库已完成 Vue 项目骨架落地，并将原有 demo 迁移为组件化页面；下一步可继续引入 UI 组件库与图表库。

### 2.2 后端（业务服务）

- 推荐：**Java 17 + Spring Boot 3.x**
- API 风格：REST
- 鉴权（后续阶段）：JWT 或基于 Session 的统一登录（二选一，结合部署形态确定）
- 接口文档：Springdoc OpenAPI（Swagger UI）
- 数据访问：MyBatis-Plus 或 Spring Data JPA（二选一，后续确定）
- 数据库：MySQL 5.7（现有环境）
- 缓存：Redis（现有环境，可选启用）

### 2.3 部署与运维

- Web 入口：Nginx（现有环境）
- 后端进程：systemd（建议）
- 前端部署：`npm run build` 后产物由 Nginx 托管（建议）
- 日志：journalctl（后端）+ Nginx access/error log
- 任务/定时：Spring Scheduler / XXL-Job（后续按需要引入）

## 4. 开发路线（里程碑 / 交付物）

### M0：演示页面阶段（已完成）

- 交付物：
  - `docs/01-需求清单.md`（待办事项模块需求）
  - `frontend/todo-demo.html`（静态演示页）
- 验收点：
  - 三大板块上下排布
  - 内容超出高度可滚动
  - 板块 1：处理按钮点击后条目消失
  - 板块 2：只展示当日变化项（演示数据体现该规则）
  - 板块 3：柱状图对比（当日/七日平均/上周同日）

### M1：前端工程化 + Mock 数据（进行中）

- 目标：把静态 demo 迁移为 Vue 工程，页面与组件可扩展、可维护
- 已完成：
  - `frontend/` 已落地 Vue/Vite 项目骨架
  - 页面路由（工作台）
  - 组件拆分（上新信息列表/竞品变化卡片/数据图表组件）
  - Mock 数据层（无需后端即可演示）
  - 已执行 `npm install`
  - 已执行 `npm run build`
- 待完成：
  - 启动本地开发服务并继续交互细化
  - 引入 UI 组件库与图表库
  - 补充页面状态管理与接口层约定
- 验收点：
  - 功能等价于静态 demo
  - 代码结构清晰，方便后续接接口

### M2：后端骨架 + 数据库落地（下一阶段）

- 目标：建立后端工程、数据库结构、并提供最小可用 API
- 交付物：
  - Spring Boot 项目
  - MySQL 数据库初始化脚本
  - API：
    - 竞品上新列表 / 处理动作
    - 竞品监控变化列表
    - 运营数据概览（按产品维度）
  - Swagger/OpenAPI 文档

### M3：联调与服务器内网部署（先内网可用）

- 目标：在服务器上实现“前端 + 后端 + 数据库”最小闭环
- 部署建议：
  - 后端监听 `127.0.0.1:8080`
  - Nginx 托管前端静态文件
  - Nginx 反代 `/api` 到后端

### M4：外网访问与安全（可选阶段）

- 域名/HTTPS、访问控制
- 登录鉴权、权限体系、操作日志
- 数据备份、监控告警
