# 需求清单

## 一、当前实现范围

当前阶段只实现 **待办事项模块**。

说明：
- 本阶段仅做模块需求定义与页面演示
- **暂不考虑数据获取**、接口联调、数据库读取与真实权限体系
- 系统未来会扩展为多模块系统，当前仅聚焦工作台中的待办事项能力

## 二、业务场景

运营人员打开系统主页后，选择 **工作台**，页面展示该运营 **今日待办事项**。

工作台中的待办事项分为 3 个板块，采用 **上下排列** 的方式展示在同一页面中；当内容超出页面高度时，页面允许 **上下滚动**。

## 三、待办事项模块需求

### 1. 板块一：竞品上新信息

展示内容：
- 竞品上新产品信息
- 上新产品的 **快照主页链接**

每条信息包含 3 个操作按钮：
- 信息推送
- 追踪
- 忽略

交互要求：
- 用户点击任意一个按钮后，**该条信息立即在页面中消失**

### 2. 板块二：竞品监控

前提：
- 数据库中存在多个已追踪竞品
- 每个竞品会记录多维信息，包括但不限于：
  - 价格
  - 销量
  - 活动
  - 图片
  - 文案
  - 当前销售排名
  - 其他相关信息

展示要求：
- 待办事项中需要 **突出展示该竞品当日发生的信息变化**
- 如果某个竞品 **当日无变化**，则 **不在工作台展示**

### 3. 板块三：运营数据

展示对象：
- 该运营名下 **每个产品** 的当日运营数据

展示内容分为三个部分：
- 销售数据
- 流量数据
- 广告数据

图表要求：
- 每个部分均使用 **柱状图** 展示对比结果
- 每个柱状图需要对比以下三个维度：
  - 当日数据
  - 七日平均数据
  - 上周同日数据

## 四、当前阶段输出要求

当前阶段需要完成：
- 明确记录上述需求
- 生成一个 **待办事项模块演示页面**
- 演示页面以静态方式展示页面结构、信息排布与核心交互

## 五、非目标

当前阶段暂不处理：
- 真实数据来源
- 接口设计与联调
- 数据库存取实现
- 登录认证
- 权限控制
- 消息推送落地能力
